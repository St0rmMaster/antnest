# Changelog

Все значимые изменения в справочнике инфраструктуры документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [1.0.0] - 2024-12-19

### Добавлено
- Создан универсальный справочник инфраструктуры
- Добавлен регламент ведения справочника
- Созданы шаблоны карточек для хостов, контейнеров и сервисов
- Добавлены примеры заполненных карточек
- Создано руководство по быстрому старту
- Определены критерии готовности (Definition of Done)
- Добавлен глоссарий основных терминов

### Миграция данных из личного справочника
- Перенесены данные хоста Proxmox VE (pve) с полной конфигурацией железа
- Созданы карточки для всех LXC контейнеров (110-160)
- Добавлены карточки для всех сервисов: PostgreSQL, nginx-proxy, n8n, Kimai, Taiga, Telegram-боты, MCP bridge, Nextcloud
- Документированы все известные "грабли" и решения
- Созданы детальные runbook'и для каждого сервиса
- Добавлен план миграции секретов в Vault
- Создана дорожная карта развития инфраструктуры

### Структура проекта
- `README.md` — основной справочник со всеми карточками
- `templates/` — шаблоны для создания новых карточек
- `examples/` — примеры заполненных карточек
- `docs/` — дополнительная документация

### Шаблоны карточек
- `templates/host-template.md` — шаблон для хостов
- `templates/container-template.md` — шаблон для контейнеров/ВМ
- `templates/service-template.md` — шаблон для сервисов/ботов

### Примеры
- `examples/example-host.md` — пример карточки Proxmox VE хоста
- `examples/example-container.md` — пример карточки LXC контейнера
- `examples/example-service.md` — пример карточки Telegram бота

### Документация
- `docs/quick-start.md` — руководство по началу работы
- `docs/changelog.md` — история изменений (этот файл)

## Как добавить запись в Changelog

При внесении изменений в справочник добавляйте записи в следующем формате:

```markdown
## [Версия] - YYYY-MM-DD

### Добавлено
- Новая функциональность

### Изменено  
- Изменения в существующей функциональности

### Исправлено
- Исправления ошибок

### Удалено
- Удаленная функциональность
```
