# Пример карточки хоста

### Хост — Proxmox VE Production

**Назначение:** Основной хост виртуализации для продакшн-среды, разворачивание контейнеров и ВМ для веб-сервисов и баз данных

**Железо:** 
- CPU: Intel Xeon E5-2680 v4 (14 ядер, 28 потоков)
- RAM: 128GB DDR4 ECC
- Диски: 
  - 2x 480GB SSD RAID1 (система)
  - 4x 2TB NVMe RAID10 (данные ВМ)
  - 2x 8TB HDD RAID1 (бэкапы)

**Сеть:** 
- Bridge: vmbr0 (основной, привязан к eth0)
- Bridge: vmbr1 (изолированная сеть для тестов)
- VLAN: 100 (продакшн), 200 (staging), 300 (мониторинг)
- Внешний IP: 203.0.113.10/24

**Хранилища:** 
- local: системные диски ВМ/контейнеров (480GB SSD)
- local-lvm: данные ВМ/контейнеров (8TB NVMe RAID10)
- backup-hdd: локальные бэкапы (16TB HDD RAID1)
- nfs-backup: удалённые бэкапы (NFS на 192.168.1.200)

**Бэкапы VM/LXC:** 
- Инструмент: Proxmox Backup Server + vzdump
- Расписание: ежедневно в 02:00 (полные), каждые 4 часа (инкрементальные)
- Хранение: локально 7 дней, на NFS 30 дней, в облаке 90 дней

**Обслуживание:** 
- Обновления PVE: первая суббота месяца в 01:00-04:00
- Проверка состояния дисков: еженедельно (smartctl)
- Проверка логов: ежедневно (journalctl, /var/log/pve/)
- Тестирование бэкапов: ежемесячно

**Известные грабли:**
- При обновлении ядра иногда не поднимаются сетевые интерфейсы — перезагрузка решает
- NVMe диски греются выше 70°C под нагрузкой — следить за температурой
- После обновления PVE проверить работу Proxmox Backup Server
- При миграции ВМ между нодами проверить совместимость CPU флагов

**Обновлено:** 2024-12-19
